# AI-in-SDLC Self-Evaluation Scorecard

**Prompting / Directing / Orchestrating / Engineering / Pioneering** — each level above Prompting represents a meaningful shift in how you work — not just how much you use AI.

The Coach uses this rubric to place you based on your actual recent practice — not self-assessment.

| Dimension | Prompting | Directing | Orchestrating | Engineering | Pioneering |
|---|---|---|---|---|---|
| **Workflow & Tooling** | Single-shot prompts with a context file. AI used reactively, one task at a time. No structured agentic workflows, Skills, MCP servers, or subagents yet. | CLAUDE.md set up for at least one project with stack context and conventions. Extended multi-turn sessions — Claude collaborates through a full task, not just one question. Understands what Claude can read (files, tools, bash). Asks for a plan before code. Knows when to /clear and when to summarise context. | Agentic workflow orchestrating analysis, coding, and testing. MCP connected for GitHub PRs and Figma. Prompted subagents used for code review and QA. Worktrees and `CLAUDE.md` / `AGENTS.md` in regular use. | Specialized agent teams with defined roles (architect, reviewer, test agent). Custom MCP server built for an internal service or API. Prompt caching applied on large-context workflows. | Runs 2–4 concurrent worktrees with dedicated agent sessions per branch. Staged workflow for large features: exploration → design doc → parallel subagent implementation + TDD. Custom Skills replace manual clipboard prompts for every repeatable task. |
| **QA** | Asks AI to write tests after implementation is done. Tests cover the same cases that would have been written manually. No test plan generated from story requirements. | Claude is in the review loop — asks it to explain failures, suggest missed edge cases, and review diffs before raising a PR. Test runs still manual. No test plan generated yet. | Generates test plan from story before implementation. Uses subagents for QA runs. AI-assisted CI failure triage before manual investigation. | Agentic Playwright testing. Scripts to call APIs as part of integration flows. iOS/Android MCP for automating UI flows. Test plan created *and* peer-reviewed before a story begins. | End-to-end agentic test pipelines: TDD skill and implementation subagent run in parallel, with Playwright screenshots or test results used as verification before any commit. Owns and defines team QA benchmarks. |
| **Skills & Community** | Techniques are personal — not yet written up or shared. AI use is invisible to teammates. | Tracks which prompting patterns work and which don't — personally, not yet shared. May have tried one technique from a course or post. AI use is still private. | Shares techniques informally (Slack, PR notes, demos). Writes at least one internal post per month — including what didn't work. Reviews code using agentic tools (e.g. Agentic Review). | Authors a Skill specific to a service, coding standard, or framework — checked in and approved by an L4 dev. Contributes to shared team `AGENTS.md` or `CLAUDE.md`. | Skills adopted across teams. Writes honestly about their AI journey over time — including tradeoffs and what they had to unlearn. That writing becomes a reference others point to. |
| **Leadership & Adoption** | AI usage is personal and not yet visible to teammates. | AI use is deliberate but personal. May have mentioned a useful technique to a teammate in passing, but no structured sharing yet. | AI usage is visible — shares techniques in PRs, Slack, demos, pair sessions. | Facilitates structured knowledge transfer: workshops, walkthroughs, or team sessions on a specific workflow. | Gets 1–2 engineers to actively adopt a new way of working (worktrees, parallel subagents, staged workflows). Leads adoption of emerging capabilities. Writing about the journey becomes a team reference. |
